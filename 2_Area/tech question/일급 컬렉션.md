### 일급 컬렉션이란

- Collection을 감싸는 클래스를 만들고, 해당 클래스에서 컬렉션과 관련된 모든 비즈니스 로직을 관리하는 패턴
- 컬렉션 하나 외에 다른 필드를 가지지 않는 것이 일반적이다.

```java
public class Orders {
    private final List<Order> orders;

    public Orders(List<Order> orders) {
        validate(orders); // 생성 시점에 검증 수행
        this.orders = orders;
    }

    public void add(Order order) {
        if (order == null) {
            throw new IllegalArgumentException("Order cannot be null");
        }
        orders.add(order);
    }

    public List<Order> getAll() {
        return Collections.unmodifiableList(orders); // 불변 반환
    }

    public double getTotalAmount() {
        return orders.stream()
                .mapToDouble(Order::getAmount)
                .sum();
    }
}
```

### 일급 컬렉션을 사용하는 이유

1. 비즈니스에 특화된 이름 부여
	 - `List<Order>` 라는 타입만으로는 이 컬렉션이 어떤 맥락에서 사용되는지 명확히 알기 어렵다. 하지만 Orders, DaisoItems 처럼 일급 컬렉션을 사용하면 비즈니스에 특화된 이름을 부여할 수 있다.

2. 로직의 캡슐화와 중복 제거
	- 상태와 행위를 한곳에서 관리할 수 있다.

```java
// Before
List<Item> daisoItems = new ArrayList<>();

// 아이템을 추가하는 곳마다 검증 로직이 퍼짐 (중복 발생)
if (item.isPriceHigherThan(5000)) {
    throw new IllegalArgumentException("5000원이 넘는 물건은 진열할 수 없습니다.");
}
daisoItems.add(item);
```

```java
// After
public class DaisoItems {
    private final List<Item> items;

    public DaisoItems() {
        this.items = new ArrayList<>();
    }

    public void addItem(final Item item) {
        // 검증 로직이 클래스 내부로 응집됨
        if (item.isPriceHigherThan(5000)) {
            throw new IllegalArgumentException("5000원이 넘는 물건은 진열할 수 없습니다.");
        }
        items.add(item);
    }
}
```

3. 컬렉션의 불변성 보장
- 외부로 컬렉션을 노출할 때 불변성을 보장할 수 있다.
